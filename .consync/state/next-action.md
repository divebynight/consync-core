GOAL

Run a short real-feeling desktop session using the full loop:
search → inspect → reveal

This is an observational package focused on identifying the next smallest friction point or missing piece in the workflow.

TRIAL TO RUN

Simulate a real usage moment:

1. Open desktop shell
2. Choose a root (nested-anchor fixture or something similar)
3. Enter a theme query
4. Run search
5. Scan results
6. Select a result
7. Inspect detail panel
8. Reveal in Finder
9. Mentally continue: “what would I do next?”

EVALUATION FOCUS

Pay attention to:

- does anything feel slow or awkward?
- do you want to click something that doesn’t exist?
- do you want more info before revealing?
- do you want to do something *after* revealing?
- does the UI feel like it supports your thinking?

IMPORTANT

- Do NOT fix anything in this package
- Do NOT add features
- Identify ONE friction point only

SUCCESS CONDITION

- either:
  A) the flow feels clean and the next step is obvious
  B) one clear next friction is identified
