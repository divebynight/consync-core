# State Hierarchy (v1)

## Filesystem
The filesystem is the source of truth for the current system.

## Execution State
`state/handoff.md` holds the latest execution state for the current workflow.

## Historical State
Future historical state may be recorded in `artifacts/work-log.md`.

## Planning State
Future planning state may be recorded in `artifacts/feature-map.md`.

## Note
Any non-filesystem layer is derived, replaceable, or reconstructable.
